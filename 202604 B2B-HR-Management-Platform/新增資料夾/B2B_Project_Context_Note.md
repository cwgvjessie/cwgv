## 📅 已完成里程碑

### 1. 核心商業邏輯確立（PRD 規格）
- 文件：`HR Admin Dashboard.html`
- **名額管控**：整批阻擋防呆；未開通仍佔用名額；權益重疊採聯集不累加
- **數據雙軌**：行政狀態即時 / 行為數據 T+1
- **有效閱讀門檻**：滑動 > 75% + 停留秒數 ≥ 文章字數 ÷ 800 × 30（前端即時計算）
- **隱私**：勤學榜 Top 10 僅人資後台可見，不對學員公開
- **移除**：立即釋出名額，歷史數據保留於企業報表
- **停用**：
  - 可指定生效日（過去/今日 → 立即；未來日期 → 排程停用）
  - 停用後名額立即釋出（或排程日釋出）
  - 已停用者可由人資「重新開通」（需有剩餘名額）
- **停用 vs 移除差異**：停用保留學員在名單中；移除從名單消失
- **合約到期預警**：30 / 14 / 7 天三波
- **勤學榜**：人資不可匯出；匯出名單不含勤學榜排名
- **SA 代入 Dashboard**：不記錄 Action Log
- **恢復停用**：恢復後不需重新發開通信（帳號仍為已開通狀態）

### 2. 高保真首頁原型（HR Dashboard Prototype.html）
- 技術：HTML + Tailwind CSS + Chart.js v4.4.0 + wordcloud2.js
- 版型：Top Navbar（slate-900 sticky）取代側欄；行動版漢堡選單
- navbar「學員名單管理」已連結至 `HR Member List.html`
- **版面順序（上→下）**：
  1. KPI 列（席次/開通率/閱讀時長/到期）
  2. 每日閱讀 PV 趨勢（Chart.js 折線，7/14/30 切換）
  3. [活躍時間熱度圖 | 員工興趣：文章分類（文字雲）]
  4. [全站熱門文章 Top 10 | 透明勤學榜 Top 10]
- 熱度圖：hover tooltip 顯示時段＋次數，顏色漸層（灰→藍→靛→紫→粉→紅）
- 勤學榜/熱門文章/文字雲/PV 圖：均支援 7/14/30 天切換

### 3. 學員名單管理頁（HR Member List.html）
- **欄位**：姓名、信箱、部門、職稱、狀態、開通日期、最後登入、有效閱讀篇數、操作
- **狀態類型**：
  - 已開通（綠）
  - 未開通（琥珀）— 佔用名額，尚未啟用
  - 停用中（紅）— 名額已釋出
  - 排程停用（橙）— 仍佔用名額，到期日自動釋出
  - 待接替（紫）— 名額由被接替者佔用（共 1 名額）
- **KPI 小卡**：總名額 / 已開通 / 未開通（佔用名額）/ 停用（名額已釋出）
- **名額計算**：active + succession + scheduled_suspend + pending 計入使用中；suspended 不計
- **篩選**：搜尋（姓名/信箱）/ 狀態 / 部門
- **批次操作**：重新發送開通信、停用、移除（釋出名額）
- **行操作對照表**：

| 狀態 | 可用操作 |
|------|---------|
| 已開通（無接替） | 設定接替、停用、移除 |
| 已開通（有接替排程） | 停用、移除 |
| 未開通 | 重發信、停用、移除 |
| 停用中 | 重新開通（需有名額）、移除 |
| 排程停用 | 取消排程、移除 |
| 待接替 | 取消接替 |

- **停用 Modal**：
  - 含日期選擇器（留空/今日前 → 立即；未來日期 → 排程）
  - 即時停用：名額立即釋出
  - 排程停用：名額於生效日釋出，姓名欄顯示橙色「排程停用｜日期」
- **權益接替排程**：
  - 從「已開通」學員行操作「設定接替」觸發
  - 填入接替人資料＋生效日，共 1 名額（不額外佔用）
  - 生效日系統自動切換；待接替者顯示「接替 XXX｜日期 生效」
- **新增學員 Modal**：支援「此為權益接替排程」toggle
- **批次匯入 Modal**：CSV 上傳、重複信箱防呆、範本下載
- **匯出**：匯出全體學員名單，不含勤學榜排名
- **Toast**：每次操作後底部彈出確認訊息（3 秒消失）
