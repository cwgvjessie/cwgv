# 圖片尺寸縮放工具 — 產品規格文件

**版本**：v1.1.0
**日期**：2026-03-18
**狀態**：Released

---

## 1. 產品概述

一款純前端的圖片縮放工具，提供兩種使用模式：

- **純計算模式**：手動輸入原始尺寸，即時計算等比例縮放後的目標尺寸，無需上傳圖片。
- **圖片縮放模式**：上傳圖片後，依設定尺寸縮放並下載結果圖。

全程在瀏覽器本機執行，圖片不離開使用者裝置。

---

## 2. 使用者故事

| ID | 角色 | 需求 | 驗收條件 |
|----|------|------|----------|
| US-01 | 使用者 | 手動輸入原始長寬，作為計算基準 | 原始尺寸欄位可直接輸入，不需上傳圖片 |
| US-02 | 使用者 | 輸入目標寬，自動算出對應的高 | 高度欄位即時更新，維持原圖比例 |
| US-03 | 使用者 | 輸入目標高，自動算出對應的寬 | 寬度欄位即時更新，維持原圖比例 |
| US-04 | 使用者 | 輸入縮放比例，自動算出目標長寬 | 長寬欄位即時更新，數值 = 原圖 × 比例 |
| US-05 | 使用者 | 上傳圖片並依目標尺寸縮放後下載 | 下載檔案尺寸符合目標長寬 |
| US-06 | 使用者 | 上傳圖片後自動帶入原始尺寸 | 原始寬高欄位自動填入，無需手動輸入 |

## 3. 功能規格

### 3.1 原始尺寸輸入模組

| 欄位 | 說明 |
|------|------|
| 原始寬度 | 手動輸入，或上傳圖片後自動填入 |
| 原始高度 | 手動輸入，或上傳圖片後自動填入 |

**行為規則**
- 兩欄位皆可直接手動輸入（純計算模式）。
- 上傳圖片後自動覆寫為圖片實際尺寸。
- 原始尺寸變更時，目標尺寸依當前比例同步重算。
- 數值限制：正整數，最小值 1。

---

### 3.2 目標尺寸計算模組

三個欄位（寬度、高度、縮放比例）彼此即時聯動，永遠維持等比例。

#### 3.2.1 輸入模式一：長寬直接輸入

| 欄位 | 說明 |
|------|------|
| 寬度（Width） | 輸入目標寬度（px），自動計算對應高度與比例 |
| 高度（Height） | 輸入目標高度（px），自動計算對應寬度與比例 |

**等比例計算公式**

```
目標高度 = round( 原始高度 × (目標寬度 / 原始寬度) )
目標寬度 = round( 原始寬度 × (目標高度 / 原始高度) )
```

#### 3.2.2 輸入模式二：縮放比例

| 欄位 | 說明 |
|------|------|
| 縮放比例（Scale） | 輸入百分比，例如 50 代表縮小為 50%，200 代表放大至 200% |

**等比例計算公式**

```
目標寬度 = round( 原始寬度 × (比例 / 100) )
目標高度 = round( 原始高度 × (比例 / 100) )
```

**共用行為規則**
- 使用者編輯任一欄位，其餘兩欄即時更新。
- 數值限制：正數，大於 0（寬高最小值 1 px）。
- 支援小數點（比例欄位精確至小數一位）。
- 原始尺寸未填入時，目標尺寸欄位不觸發計算。
- 輸入無效值時，欄位顯示紅色邊框並提示錯誤訊息。

---

### 3.3 圖片上傳模組

| 項目 | 規格 |
|------|------|
| 支援格式 | JPG / JPEG、PNG、WebP、GIF（靜態）、BMP |
| 上傳方式 | 點擊選檔 或 拖曳上傳（Drag & Drop） |
| 檔案大小上限 | 20 MB |
| 上傳後顯示 | 預覽縮圖、原始寬高（px）、檔案格式、檔案大小 |
| 上傳後行為 | 自動填入原始尺寸欄位，目標尺寸重設為 100% |

**錯誤處理**

| 情境 | 提示訊息 |
|------|----------|
| 格式不支援 | 「不支援此格式，請上傳 JPG / PNG / WebP / GIF / BMP」 |
| 超過大小限制 | 「檔案大小超過 20MB，請選擇較小的檔案」 |
| 上傳失敗 | 「圖片讀取失敗，請重新嘗試」 |

---

### 3.4 縮放與下載模組

> 需上傳圖片後才可使用，純計算模式不提供下載。

#### 3.4.1 縮放執行
- 使用者設定好目標尺寸後，點擊「縮放並下載」按鈕。
- 縮放在瀏覽器端透過 Canvas API 完成，不上傳至伺服器。
- 縮放演算法：雙線性插值（Bilinear Interpolation）以維持圖片品質。

#### 3.4.2 下載設定

| 項目 | 規格 |
|------|------|
| 輸出格式 | 預設與原始格式相同；提供下拉選單可選擇 JPG / PNG / WebP |
| 輸出品質 | 僅 JPG / WebP 顯示品質滑桿（1–100，預設 90）；PNG 不顯示 |
| 檔案命名 | `{原始檔名}_{目標寬度}x{目標高度}.{副檔名}` |
| 下載方式 | 觸發瀏覽器原生下載 |

#### 3.4.3 執行流程

```
[純計算模式]
  手動輸入原始尺寸
    → 輸入目標尺寸 / 比例
    → 即時顯示計算結果

[圖片縮放模式]
  上傳圖片
    → 自動填入原始尺寸
    → 輸入目標尺寸 / 比例
    → 點擊「縮放並下載」
    → Canvas 繪製縮放後圖片
    → 觸發下載
```

## 4. UI / UX 規格

### 4.1 頁面佈局（單頁）

```
┌─────────────────────────────────────┐
│           圖片縮放工具               │
├─────────────────────────────────────┤
│  [上傳區域 / 拖曳區域]              │
│  （上傳後顯示預覽 + 原始尺寸資訊）  │
├─────────────────────────────────────┤
│  原始尺寸                           │
│  原始寬 [____] px × 原始高 [____] px│
├─────────────────────────────────────┤
│  目標尺寸                           │
│  寬度 [____] px  ⇔  高度 [____] px │
│  縮放比例 [____] %                  │
├─────────────────────────────────────┤
│  輸出格式 [JPG ▼]  品質 [90 ====]  │  ← 上傳圖片後才顯示
│                                     │
│        [縮放並下載]                 │  ← 無圖片時 disabled
└─────────────────────────────────────┘
```

### 4.2 互動回饋

| 情境 | 行為 |
|------|------|
| 未填原始尺寸 | 目標尺寸欄位不觸發計算 |
| 未上傳圖片 | 輸出設定隱藏；「縮放並下載」顯示但 disabled |
| 上傳圖片後 | 輸出設定顯示；按鈕在尺寸有效時 enabled |
| 縮放執行中 | 按鈕顯示 Loading 旋轉圖示 |
| 輸入無效值 | 欄位紅框 + 錯誤提示文字 |

## 5. 非功能需求

| 類別 | 規格 |
|------|------|
| 執行環境 | 純前端（瀏覽器），無需後端伺服器 |
| 隱私安全 | 圖片不離開使用者裝置，全程本地處理 |
| 瀏覽器支援 | Chrome 90+、Firefox 88+、Safari 14+、Edge 90+ |
| 響應式設計 | 支援桌機與平板（最小寬度 768px） |

## 6. 技術實作

| 項目 | 方案 |
|------|------|
| 框架 | 原生 HTML / CSS / JavaScript（單一 index.html） |
| 縮放實作 | `<canvas>` + `drawImage()` |
| 下載實作 | `canvas.toBlob()` + `URL.createObjectURL()` |
| 樣式 | 原生 CSS |
| 部署 | GitHub Pages（靜態檔案，無需建置步驟） |

---

## 7. 範圍外（Out of Scope）

- 批次多圖上傳與下載
- 非等比例縮放（長寬比固定，不可解鎖）
- HEIC / AVIF 等進階格式支援
- 圖片裁切（Crop）
- 圖片旋轉、翻轉
- 浮水印功能
- 後端儲存或雲端同步
- GIF 動態幀縮放
